# Traceability Matrix: SDD → HLRs → LLRs → Tests

**Version:** 2.0
**Date:** 4/21/2026
**Author(s):** John Anderson

This document provides the bi-directional traceability between
the four layers of the `vgp` requirements/design/verification
stack:

1. [Software Design Document](SDD.md) (SDD) — architecture and
   module design.
2. [High-Level Requirements](HLRs.md) (HLR) — externally
   observable behaviour the application must provide.
3. [Low-Level Requirements](LLRs.md) (LLR) — per-function
   behavioural contracts that implement the HLRs.
4. [Software Test Plan](STP.md) and the test artefacts
   under [test/](../test/) — cmocka unit tests and integration
   tests that verify each LLR.

It is generated by `tools/render_doc.py` from the
`<traces>` blocks attached to every HLR, LLR, and test in
[doc/Project.xml](Project.xml). The current snapshot covers
**45 HLRs**, **124 LLRs**,
and **120 tests** across
**6 test files**. Every reference
below is a clickable link to the artifact in its source document.

## 7. Conventions

*   Each HLR carries a `*Trace:*` line in [HLRs.md](HLRs.md)
    listing the SDD sections it implements.
*   Each LLR carries a `*Trace:*` line in [LLRs.md](LLRs.md)
    listing the HLRs it implements (with the HLR's short name).
*   Each test function under [test/](../test/) carries a
    doc-comment block citing the LLR(s) and HLR(s) it verifies,
    mirrored into [Project.xml](Project.xml) as `<traces>`.
*   The [Software Test Plan](STP.md) is the canonical
    catalogue of test purposes; this matrix is the canonical
    cross-reference.
*   This matrix is a snapshot. After modifying any of the four
    upstream documents or the test sources, regenerate it via
    `python3 tools/render_doc.py tools/templates/Traceability.md.j2 Traceability --out doc/Traceability.md`.
